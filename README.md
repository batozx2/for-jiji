# For Soujoud ❤️

A static, mobile-first romantic one-page website. It works on GitHub Pages with no backend and no build step.

## Replace Photos

Open `assets/` and replace:

- `photo1.jpg`
- `photo2.jpg`
- `photo3.jpg`
- `photo4.jpg`
- `photo5.jpg`
- `photo6.jpg`
- `photo7.jpg`
- `photo8.jpg`
- `photo9.jpg`
- `photo10.jpg`

Keep the same filenames so the gallery works without editing code. Portrait photos look best on phones.

## Replace Music

Replace `assets/music.mp3` with your own MP3 file and keep the exact filename `music.mp3`.

The page already points to `assets/music.mp3`, loops it, attempts autoplay when the browser allows it, and shows a play button when autoplay is blocked.

## Edit The Letter And Reasons

- Edit the letter inside `index.html` in the `LETTER SECTION`.
- Edit `Reason #1` through `Reason #15` inside the `REASONS I LOVE YOU SECTION`.

## Deploy To GitHub Pages

1. Create a new GitHub repository.
2. Upload `index.html`, `style.css`, `script.js`, and the `assets/` folder to the repository root.
3. Go to repository `Settings`.
4. Open `Pages`.
5. Under `Build and deployment`, choose `Deploy from a branch`.
6. Select branch `main` and folder `/root`.
7. Save, then wait for GitHub to publish the site.

Your page will be available at the GitHub Pages URL shown in the Pages settings.
